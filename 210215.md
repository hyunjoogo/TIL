# HTML

HTML5는 구성에 중심을 두었다. 각 요소가 독립된 형태를 유지하면서 아울러 이 요소를 모아 완성된 형태가 되는 구성에 중심을 두었다.

DOM 트리 구조를 가진 상태에서 구성이라는 개념을 추가하여 사상을 재정의한 것.

## 스크립팅

### `<script>` 엘리먼트

HTML5는 스크립트 실행 시점을 세가지 모드(mode)로 제공한다.

- 실행할 수 있는 상태가 되었을 때 비동기로 실행 (async)
- 다큐먼트를 파싱한 후에 실행 (defer)
- 스크립트를 만나는 시점에 실행 ( 둘 다 안 쓰면)

다큐먼트를 파싱하다가 `<script>` 엘리먼트를 만나면 서버에서 JS파일을 가져오게 되고 가져온 후에는 JS코드를 실행한다. JS가 실행완료할 때까지 그 아래에 작성한 모든 엘리먼트는 파싱을 하지 못하고 기다려야 한다.

async와 defer 차이

`defer`를 DOM 전체가 필요한 스크립트나 실행 순서가 중요한 경우에 적용합니다.

`async`는 방문자 수 카운터나 광고 관련 스크립트같이 독립적인 스크립트에 혹은 실행 순서가 중요하지 않은 경우에 적용합니다.

## Section

body, h1~h6, hgroup, header, footer, section, nav, article, aside, address

HTML5의 사상인 구성을 위한 엘리먼트. 독립된 구성이 키워드

### <section>

책의 목차, 시스템 설계서 목자, 시스템 사용자 메뉴얼 목차 정도로 생각할 수 있다.

스크립팅의 편리함을 위하거나 CSS 스타일만을 위해 사용하는 것은 부적절 (그런 목적이면 <div>)

<section> 엘리먼트는 제목과 콘텐츠로 구성된다. 즉, <h1>엘리먼트가 있어야하고 이를 보충 설명하는 <p>와 같은 엘리먼트가 있어야 한다. 아티클 성격이므로 <article> 엘리먼트가 있어야 한다.

각 엘리먼트에 <h1>엘리먼트를 넣는다면 <h1>엘리먼트가 제목이 된다.

<section> 엘리멘트는 책의 장/절을 구분하는 역할을 <h1> 엘리먼트는 장/절 제목을 표현하는 역할을 한다.

<section> 안에 <section> 작성할 수 있다.

```html
<body>
  <h1>HTML5 스터디 개최</h1>
  <!-- body엘리먼트 전체의 제목-->
  <p>HTML5 스터디를 다음과 같이 개최합니다.</p>

  <section>
    <!--일반 사항 섹션-->
    <!-- <h2> 엘리멘트를 작성하지 않는 이유는 상하구조가 아니라 구성이기 때문이다. -->
    <h1>일반 사항</h1>
    <!--일반 사항 section의 제목-->
    <p>장소 : 제1강의실</p>
    <p>시간 : 매주 토요일 오전 10시</p>
    <p>인원 : 10명 정원제</p>
  </section>
  <section>
    <!-- 신청자 섹션 -->
    <h1>신청자</h1>
    <!--신청자 section의 제목-->
    <ul>
      <li>홍길통</li>
      <li>나으뜸</li>
    </ul>
  </section>
</body>
```

### 아웃라인 생성

아웃라인의 정확성 여부를 체크하기 위한 검사기 ([https://gsnedders.html5.org/outliner/](https://gsnedders.html5.org/outliner/))

```html
<h1>마우스</h1>

<h2>클릭</h2>

<h3>왼쪽 버튼 클릭</h3>

<h3>오른쪽 버튼 클릭</h3>

<h2>드래그</h2>
```

아웃라인 검사기를 통한 체크

목차 같은 느낌이 나게 <h?>엘리먼트를 작성하는 것이 아웃랑인의 키워드

> 이런 작업이 왜 필요할까?
> HTML은 컨텐츠를 전달하는 것이 목적이다. 컨텐츠 전달의 효율성을 위해서 구조적인 형태가 기본이다. 원하는 자료를 빠르게 찾기 위해 제대로된 목차가 필요할 것이다.
> 그리고 컨텐츠 안에 컨텐츠를 구성하기 위함이라고 생각된다.

### 섹셔닝 루트 카테고리

blockquote, details, fieldset, figure, td

하나의 카테고리로 독립적인 아웃라인을 갖게 된다. 다른 섹션과 헤딩의 아웃라인에 영향을 미치지 않는다.

```html
<h1>마우스</h1>

<blockquote>
  <h2>클릭</h2>
</blockquote>

<h3>왼쪽 버튼 클릭</h3>

<h3>오른쪽 버튼 클릭</h3>

<h2>드래그</h2>
```

### 헤딩과 섹션

<h?>엘리먼트로 구성하는 아웃라인보다 HTML5가 요구하는 더 구성진 형태.

```html
<h1>마우스</h1>

<h2>클릭</h2>

<h3>왼쪽 버튼 클릭</h3>

<h3>오른쪽 버튼 클릭</h3>

<h2>드래그</h2>
```

<h1>마우스</h1>는 책 목차의 장이다. (상위 카테고리)

<h1>클릭</h1>과 <h1>드래그</h1>은 책 목차의 절이다. (마우스의 하위 카테고리)

### <header>

헤딩 콘텐츠 그룹을 표현한다.

섹셔닝 콘텐츠가 아니라 그룹화를 위한 엘리먼트이다.
(엘리먼트 안에 <section>엘리먼트가 들어오지 않는다)

### <footer>

다큐먼트 작성자, 관련 문서의 링크, 저작권자 등과 같은 콘텐츠를 작성한다.

<footer>엘리먼트는 다수를 작성할 수 있다.
다수로 작성했을 경우 가장 가까운 선조 섹셔닝 콘텐츠의 푸터가 된다. (parentNode의 parentNode)

섹셔닝 콘텐츠가 아니라 그룹화를 위한 엘리먼트이다.
(엘리먼트 안에 <section>엘리먼트가 들어오지 않는다)

```html
<body>
  <article>
    <h1>HTML5 세미나 개최</h1>
    <p>다음과 같이 HTML5 세미나를 개최합니다.</p>
    <p>지난번 세미나 동영상은 <a href="#">여기</a>를 참조하세요.</p>
    <footer>
      <p>저작권자 &copy; 2010 김현주</p>
      <!--<article>의 footer -->
    </footer>
  </article>

  <footer>
    <!--<body>의 footer -->
    <nav>
      <p>
        <a href="#">고객센터</a>
        <a href="#">사이트맵</a>
        <a href="#">찾아오시는 길</a>
      </p>
    </nav>
  </footer>
</body>
```

### <nav> 내비게이션 링크 리스트

페이지의 가독성과 접근성을 향상시키려는데 있다.

주로 <li> 엘리먼트와 <a> 엘리먼트를 사용하여 메뉴를 작성하는 형태

링크를 가로 또는 세로로 놓을지는 CSS에서 결정한다.

페이지 하단에 작성하는 페이지 링크는 <footer>엘리먼트로 작성한다.

### <article>독립적인 콘텐츠

자체적으로 완성된 콘텐츠.

포럼, 잡지/신문의 아티클, 블로그, 본문의 댓글, 위젯, 가젯, 사용자 의견, 독립적인 콘텐츠가 해당된다.

쉽게 생각하면 <article> 엘리먼트를 복사해서 안의 내용만 조금 변경해서 하나의 형식처럼 쓸 수 있느냐이다.

### <aside> 간접적인 콘텐츠

다큐먼트의 주된 콘텐츠가 아닌 보조 콘텐츠를 작성할 때 사용한다. (엘리먼트가 삭제되어도 메인 컨텐츠에 영향이 없는 것)

기준이 애매모호하므로 단적으로 이 때 사용해야한다는 정의는 없다.
